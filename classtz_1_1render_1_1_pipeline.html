Detailed Description

Represents at least zero render-passes.

Can be used to neatly abstract away complicated render-pass sequences. Example:

  • 0: Device to render the scene into a frame (outputs to texture 'tex')
  • 1: Device to render the texture 'tex' into the window frame (Using a post-processing shader) Pipelines can have devices added to them. Clearing/Rendering a pipeline will invoke the same operation on all devices in chronological order.
