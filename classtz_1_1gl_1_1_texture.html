Detailed Description

Used to store image data in VRAM.

Can be attached to a shader to bind a texture to a given tz::gl::Object.

◆ set_data()

template<PixelType PixelT>
void tz::gl::Texture::set_data ( const tz::gl::Image< PixelT > &  image)

Replace the existing underlying image data with some new data.

Any data in the data-store beforehand will be discarded. Note: If the new image data format is different or the dimensions do not match, the underlying image data store is automatically resized. Precondition: If the texture is terminal and the underlying image data store is automatically resized, this will assert and invoke UB.

Example: tz::gl::Image<tz::gl::PixelRGBA8> resolves to PixelType == tz::gl::PixelRGBA, ComponentType == std::byte.

Template Parameters
PixelTypeType of the pixel template to use. Example: tz::gl::PixelRGBA
ComponentTypeUnderlying data type of the components within the pixel.
Parameters
imageImage whose data should be uploaded to VRAM.
